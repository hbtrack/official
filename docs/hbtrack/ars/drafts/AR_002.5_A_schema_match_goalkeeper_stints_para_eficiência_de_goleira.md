# AR_002.5_A — Schema: match_goalkeeper_stints para Eficiência de Goleira

**Status**: ✅ SUCESSO
**Versão do Protocolo**: 1.1.0

## Descrição
LACUNA DETECTADA: O PRD V1.1 (seções 4.2, 5 e 14) exige a métrica 'Eficiência de Goleira' nos relatórios avançados de scout. A fórmula é: saves / (saves + goals_conceded) × 100. O schema atual permite contar saves via event_type='goalkeeper_save', MAS não permite determinar quantos gols foram concedidos POR GOLEIRA ESPECÍFICA quando há substituição de goleira durante o jogo. Não existe tabela ou coluna que registre os 'stints' de cada goleira (qual período ela estava em campo).

AÇÃO: Criar migration Alembic para adicionar a tabela match_goalkeeper_stints. Usar Python (.py), NÃO .sh/.ps1.

ESTRUTURA DA TABELA:
```sql
CREATE TABLE public.match_goalkeeper_stints (
    id uuid DEFAULT gen_random_uuid() NOT NULL,
    match_id uuid NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
    athlete_id uuid NOT NULL REFERENCES athletes(id) ON DELETE RESTRICT,
    start_period_number smallint NOT NULL,
    start_time_seconds integer NOT NULL,
    end_period_number smallint,     -- nullable = ainda em campo
    end_time_seconds integer,       -- nullable = ainda em campo
    created_at timestamptz DEFAULT now() NOT NULL,
    CONSTRAINT ck_gk_stints_start CHECK (start_time_seconds >= 0),
    CONSTRAINT ck_gk_stints_end CHECK (end_time_seconds IS NULL OR end_time_seconds >= 0),
    CONSTRAINT ck_gk_stints_period CHECK (start_period_number >= 1)
);
CREATE INDEX ix_match_goalkeeper_stints_match_id ON match_goalkeeper_stints(match_id);
CREATE INDEX ix_match_goalkeeper_stints_athlete_id ON match_goalkeeper_stints(athlete_id);
```

REGRA DE PREENCHIMENTO: Quando match_events registra event_type='substitution' e secondary_athlete_id é uma goleira (verificar match_roster.is_goalkeeper=TRUE para secondary_athlete_id), encerrar o stint anterior e criar novo para o athlete_id que entrou. O treinador também pode preencher stints manualmente no pós-jogo.

FÓRMULA HABILITADA:
goals_conceded_by_gk = COUNT de eventos adversários que resultaram em gol durante o stint = MAX(score_opponent) no fim do stint - MAX(score_opponent) no início do stint.
Alternativamente: usar as diferenças em score_opponent nos match_events durante o intervalo de tempo do stint.
