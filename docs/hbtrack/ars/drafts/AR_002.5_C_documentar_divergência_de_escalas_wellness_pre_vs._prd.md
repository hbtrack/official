# AR_002.5_C — Documentar divergência de escalas wellness_pre vs. PRD

**Status**: 🏗️ EM_EXECUCAO
**Versão do Protocolo**: 1.1.0

## Descrição
DIVERGÊNCIA DETECTADA (não-bloqueante para V1.1): PRD Seção 7 (US-002) especifica 'escala 1-5' para os campos de wellness pré-treino: humor, sono, fadiga, dores. O schema real usa escalas distintas:
- stress_level: 0-10 (proxy de 'humor', semântica inversa: alto=ruim)
- fatigue_pre: 0-10 (PRD diz 1-5)
- muscle_soreness: 0-10 (PRD diz 1-5)
- sleep_quality: 1-5 (alinhado com PRD)
- sleep_hours: numeric(4,1) em horas (não estava no PRD, mas é dado adicional útil)
Além disso, NÃO EXISTE coluna nomeada 'humor' ou 'mood' — o mapeamento UI → DB é implícito.

DECISÃO: Avaliar se a escala 0-10 é INTENCIONAL (mais granularidade clínica) ou erro de alinhamento. Não alterar o schema sem aprovação, pois dados de produção podem já existir na escala 0-10.

AÇÃO REQUERIDA (sem alterar schema):
1. Atualizar o PRD Seção 7 (US-002) para refletir a escala real: 'escala 0-10 (fatigue, stress, soreness), 1-5 (sleep_quality), horas numéricas (sleep_hours)'.
2. Atualizar o Glossário PRD (Seção 17) adicionando: 'stress_level = proxy de humor/disposição (0=muito bem, 10=muito estressado)'.
3. Criar coluna de alias no OpenAPI (stress_level descrito como 'Humor/Disposição' no schema Pydantic — via Field(alias='humor') opcional).
4. Atualizar docs/hbtrack/PRD HB TRACK.md, Seção 7, US-002, Critério 2: substituir 'escala 1-5' pela escala real de cada campo.

Nota: NENHUMA migration de schema é necessária para este item — apenas atualização de documentação e possivelmente o Pydantic field description.

## Critérios de Aceite
1) Seção 7 do PRD HB TRACK.md atualizada com as escalas reais (0-10 vs 1-5). 2) Seção 17 (Glossário) do PRD atualizada com definição de stress_level como proxy de humor. 3) Schema Pydantic wellness_pre (se existir) tem Field descriptions atualizadas refletindo os ranges reais. 4) Decisão sobre mudança de escala documentada (manter 0-10 ou migrar para 1-5) em docs/ars como registro arquitetural.
